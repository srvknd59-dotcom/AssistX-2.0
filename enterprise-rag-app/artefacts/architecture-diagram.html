System Architecture

Two things happen in this system: documents get ingested once (or whenever new ones are added), and questions get answered every time someone asks one. Both flows meet at the same vector store.
Ingestion flow Query flow Dashed = external service

Ingestion — runs once per document set

POST /ingest  ·  python ingest.py
1 Source Documents .txt / .md / .pdf in data/documents, or uploaded from the UI
→
2 chunking.py Chunk Split into ~180-word overlapping passages
→
3 OpenAI API Embed text-embedding-3-small → 1536-dim vector per chunk
→
4 vector_store*.py Elasticsearch rag_documents index · dense_vector field

Query & answer — runs on every chat message

POST /chat/send
1 React UI · :5173 Question Typed in the chat box, sent with session_id
→
2 OpenAI API Embed question Same embedding model as ingestion
→
3 vector_store*.py kNN search Top-k nearest chunks by cosine similarity
→
4 pipeline.py Build prompt Numbered context + question + system rules
→
5 OpenAI API Generate gpt-4o-mini answers only from context, cites [1] [2]
→
6 React UI Answer + sources Rendered with expandable citations
⋮ Steps 4 and 3 hit the same Elasticsearch index: {ES_INDEX_PREFIX}_documents (default rag_documents) — set in backend/.env.
